ANIMOTION

Animotion is an open-source repository that contains pre-made animation files that are free for use in personal and commercial projects. Animotion helps web developers discover, create, test, fine-tune, and implement lightweight, interactive animations for web and mobile projects.

This project was made as part of the I4G x ZURI training final project phase, we were tasked to create a website library that allows beginners perform basic web animation using normal language.

Team made up of 12 designers and 10 developers.

Role

Product Design Team Lead

Timeline

4 Weeks (August 2022)

Tools

  • Figma
  • Slack
  • Google suits

Problem Statement

The animation system is one of the most difficult subsystems to implement since it needs to be integrated with the majority of the software's basic features.
It is also one of the fundamental components of any web development. Web developers have struggled with CSS animations for a long time, so a library with all of the essential features would be a perfect answer.

Solution

  • Create a feature that allows users save data and download it at anytime.
  • Allow users to preview examples.
  • Simplify documentation to make sure a beginner can easily understand how to use the library.
  • Create an interactive interface that enables code manipulation.

EMPATHIZE PHASE

USER RESEARCH

I helped in carrying out the research, we aimed to find out what habits and motivations our users had.

  • To know how frequently developers use animation libraries.
  • To have insights on developers' experience with animation libraries.
  • To identify the barriers developers encounter while interacting with animation libraries.
  • To understand the end-to-end process of how users interact with animation libraries.
  • To know how to simplify animation libraries for developers.

USER INTERVIEW

We interviewed 7 developers to find challenges they face when using an animation library. To recruit participants, screener questions were used to identify developers who use animations via google meet and zoom calls.

COMPETITOR ANALYSIS

I was involved in carrying out the competitor analysis to help us understand what we need to make Animotion website better in terms of features. Google sheets was used in organizing our findings.

DEFINE PHASE

USER PERSONA

With data collected from interviews, a persona representing an ideal user of the website. It helped us arrive at better solution as it gave an in-depth understanding of the user goals and frustrations.

EMPATHY MAP

To empathize with the user, we came up with an empathy map for better connectivity and understanding of the user.

USER STORY

We also came up with user story, for better understanding of the user journey.

IDEA PHASE

BRAINSTORMING

All team members came together to hold a brainstorming session and came up with ideas. We discussed and voted on the ideas as well as combined some ideas.

USER FLOW

We came up with the user flow to help us understand how our user might navigate through the website.

DEVELOPING THE DESIGN

After all high fidelity mockups were designed, we handed over our designs to the development team by indicating on Github that the design tasks are finished.
We worked very closely with the front-end team to spec out any missing interactions that were not covered in the high fidelity mock up.

RESULTS AND TAKEAWAYS

We were able to successfully implement 85% of the design and delivered a presentation to the panel of judges.

Some key takeaways are :

  • Best Designs Comes From Collaboration :It was encouraging to see how each person was able to bring ideas to the table from the research to sketching and creating the final designs.
  • Putting Users First :We realised that putting the user front and center of our product makes the design process easier.
  • Evaluating Our Design To Ensure Usability : Conducting usability testing helped us iterate our designs to make it usable and useful for our users.
